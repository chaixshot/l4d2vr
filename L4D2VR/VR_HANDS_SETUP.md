# VR hands dependency setup

Run this once from a PowerShell prompt before building L4D2VR:

```powershell
powershell -ExecutionPolicy Bypass -File .\L4D2VR\setup_vr_hands_dependencies.ps1
```

The script downloads a fixed cgltf revision and ozz-animation 0.16.0, then builds Win32 Debug and Release static libraries.

After installation it automatically removes the downloaded ozz-animation source tree, the CMake build directory, temporary files and duplicate per-configuration header trees. The retained dependency layout is intentionally compact:

```text
thirdparty\
  cgltf\
    cgltf.h
  ozz-animation-install\
    win32\
      include\
        ozz\...
      lib\
        Debug\
          ozz_animation.lib
          ozz_animation_offline.lib
          ozz_base.lib
        Release\
          ozz_animation.lib
          ozz_animation_offline.lib
          ozz_base.lib
```

The `ozz_animation_offline.lib` library remains required because the plugin builds an ozz runtime skeleton from the SteamVR hand hierarchy during initialization.

The setup script also migrates and cleans the older expanded layout. If the dependency folders were already generated by an earlier patch, run the script again once; it reuses the installed files and removes unnecessary directories without downloading them again.

At startup the plugin generates a copy of the SteamVR action manifest and its default binding JSON files, adding the left/right skeleton mappings without editing your original files.

The first implementation draws independent VR hands only when `mat_queue_mode == 0`. The queued renderer needs a DXVK-side safe submission point before raw D3D hand drawing can be enabled there.
